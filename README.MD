1. Запуск приложения
   Сначала запустите приложение: mvn spring-boot:run
   Приложение будет доступно по адресу: http://localhost:8080
2. Тестирование эндпоинтов в Postman:
   Получение всех контактов
   GET http://localhost:8080/contacts
   Нажмите "Send"
   Должен вернуться пустой массив [] (пока контактов нет)
3. Перейдите на вкладку "Body"
   Выберите "raw" и "JSON"
   Введите данные:
   {
   "firstName": "Иван",
   "lastName": "Иванов",
   "phone": "+79991234567",
   "email": "ivan.ivanov@example.com"
   }
   Нажмите "Send"
   В ответе получите созданный контакт с присвоенным ID
   Получение конкретного контакта
(где 1 - это ID созданного контакта)
   Обновление контакта
PUT http://localhost:8080/contacts/1
   Вкладка "Body" → "raw" → "JSON"
   Введите обновленные данные:
4. {
   "firstName": "Петр",
   "lastName": "Петров",
   "phone": "+79997654321",
   "email": "petr.petrov@example.com"
   }
   Нажмите "Send"
Проверка изменений
GET http://localhost:8080/contacts
   Убедитесь, что контакт обновился
   Пример последовательности тестов:
   POST /contacts - создать первый контакт
   POST /contacts - создать второй контакт
   GET /contacts - получить список всех контактов
   GET /contacts/1 - получить первый контакт
   PUT /contacts/1 - обновить первый контакт
   GET /contacts/1 - проверить обновление

   Важные моменты:

   Убедитесь, что Content-Type в заголовках установлен как application/json
   При создании контакта не указывайте ID - он присвоится автоматически
   При обновлении контакт перезаписывается полностью
   Если указать несуществующий ID, получите статус 404
   После каждого запроса проверяйте статус ответа (200 - успех, 404 - не найдено).
